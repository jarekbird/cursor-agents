## Description

Add test cases for the `getTaskStatus` and `markTaskComplete` methods to verify they correctly read and update task statuses. The tests should cover reading status for existing and non-existent tasks, and marking tasks as complete.

**This task has been split into subtasks - see TASK-014.1 and TASK-014.2**

**Reference Implementation**: 
- `virtual-assistant/cursor-agents/src/services/database-service.ts` (lines 205-227) - `getTaskStatus()` and `markTaskComplete()` methods

## Current State

The `getTaskStatus` method:
- Queries `tasks` table for status by task ID
- Returns status number or `null` if task not found
- Returns `null` on error and logs the error

The `markTaskComplete` method:
- Calls `updateTaskStatus(taskId, STATUS_COMPLETE)` (status = 1)
- Is a convenience wrapper for marking tasks complete

The test file `tests/services/database-service.test.ts` exists with previous task tests.
